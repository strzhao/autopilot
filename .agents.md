# .agents.md

本文件是本仓库给 Codex 使用的项目级说明。

根目录 `AGENTS.md` 在本仓库中是已有软链接，指向 `CLAUDE.md`，因此本次兼容层不直接改它，而是通过 `.codex/config.toml` 里的 `project_doc_fallback_filenames = [".agents.md"]` 让 Codex 读取本文件。

## 基本约束

- 现有 Claude 插件源码仍是 Claude 侧的唯一真源。
- 进行 Codex 兼容工作时，优先只改 `.agents/`、`.codex/`、`codex/docs/`。
- 除非任务明确要求改 Claude 插件本体，否则不要改 `plugins/`、`.claude-plugin/marketplace.json`、`CLAUDE.md`、`README.md`。
- Codex 运行时状态、报告、临时产物统一写到 `.codex/`，不要写入 `.claude/` 的运行时文件。
- `.claude/knowledge/` 可以作为只读知识库复用。

## Repo-Local Codex Skills

本仓库提供以下 Codex 专用技能：

- `$autopilot-codex`
- `$autopilot-commit-codex`
- `$autopilot-doctor-codex`

如果用户沿用 Claude Code 的说法：

- `/autopilot ...` 对应 `$autopilot-codex`
- `/autopilot commit` 对应 `$autopilot-commit-codex`
- `/autopilot doctor` 对应 `$autopilot-doctor-codex`

## Codex 适配边界

- 不再恢复历史 `plugin-sync` 的软链接共享方案。
- 不再依赖私有 bridge、watcher、日志轮询去模拟 Claude hooks。
- Codex 只使用当前仓库中的官方能力面：
  - `.agents.md`（项目级说明 fallback）
  - `.codex/AGENTS.md`（repo-local `CODEX_HOME` 启动说明）
  - `.agents/skills`
  - `.codex/hooks.json`

## 启动方式

优先使用仓库内启动脚本：

- `codex/bin/codex-local`

等价手动方式：

- `CODEX_HOME="$(git rev-parse --show-toplevel)/.codex" codex`

这样可以在不修改全局 `~/.codex/` 配置的前提下，启用：

- repo-local `codex_hooks`
- `.agents.md` 项目说明 fallback

## 共享参考资料

Codex 技能可以把以下现有文件当作只读基线：

- `plugins/autopilot/skills/autopilot/SKILL.md`
- `plugins/autopilot/skills/autopilot-commit/SKILL.md`
- `plugins/autopilot/skills/autopilot-doctor/SKILL.md`
- `plugins/autopilot/skills/autopilot/references/`

使用这些文件时，以 Codex 技能中的覆盖规则为准，不要直接照搬其中的 Claude 专有机制。
