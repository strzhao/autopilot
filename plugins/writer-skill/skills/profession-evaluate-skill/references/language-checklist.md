# 语言精度检查清单

本文件是 writer-evaluate-skill D5 维度的详细检查清单。评价时按需加载。

---

## 一、禁止开头模式

| 禁止模式 | 问题 | 替代方向 |
|----------|------|----------|
| "本文将介绍/探讨/分析……" | 元叙述，浪费读者注意力 | 直接陈述核心事实或问题 |
| "随着……的快速发展/日益普及……" | 套话，所有文章都能用 = 没有信息量 | 用具体数据或事件开头 |
| "在当今……的背景/时代下……" | 同上 | 同上 |
| "众所周知……" | 如果真众所周知，不需要说 | 删除，直入主题 |
| "还记得……吗？" | 反问式开头，试图制造共鸣但显廉价 | 直接描述问题场景 |
| "想象一下……" | 场景构建式开头，降低信息密度 | 用真实案例替代虚构场景 |

---

## 二、禁止结尾模式

| 禁止模式 | 问题 | 替代方向 |
|----------|------|----------|
| "综上所述"、"总结一下"、"总而言之" | 重复全文，浪费篇幅 | 指向未来行动或下一步 |
| "让我们拭目以待"、"未来可期" | 空话，不传递新信息 | 给出具体的前瞻判断或行动建议 |
| "写在最后"、"最后说两句" | 口语化，降低专业感 | 直接写最后一节的内容 |
| 鸡汤式金句结尾 | 与专业技术文章调性冲突 | 以数据或行动收尾 |
| 重复全文要点的总结段 | 读者已经读完了，不需要再听一遍 | 删除，或改为 1 句前瞻 |

## 三、禁止语气词（按类型分组）

### 自我庆祝
- "我们很自豪地宣布"、"我们激动地分享" → 直接说发布了什么
- "经过团队的不懈努力" → 删除，读者不关心过程

### 情感渲染
- "令人振奋"、"激动人心" → 用数据表达影响力
- "革命性的"、"颠覆性的" → 描述具体变化是什么
- "前所未有"、"史无前例" → 给出对比数据

### 营销话术
- "game-changing"、"cutting-edge"、"行业领先"、"业界首创" → 用事实和数据替代
- "one-stop solution"、"端到端" → 列出具体覆盖的功能

### 口语化（专业文章不应出现）
- "说实话"、"说白了"、"这玩意"、"搞" → 用书面语替代
- "大白话来说" → 直接用清晰的语言，不需要声明

### 学术腔
- "笔者认为" → 直接表达观点
- "不难发现" → 直接说发现了什么
- "值得注意的是" → 直接说要注意什么

## 四、商业黑话

| 黑话 | 问题 | 替代建议 |
|------|------|----------|
| 赋能 | 模糊、过度使用 | 说清楚具体提供了什么能力 |
| 闭环 | 同上 | 说清楚从哪到哪的完整流程 |
| 抓手 | 同上 | 说清楚具体的工具或方法 |
| 深耕 | 同上 | 用"专注于"或具体时间 |
| 沉淀 | 同上 | "积累"、"记录"或具体动作 |
| 生态 | 滥用 | 除非真的是平台生态系统 |
| 矩阵 | 同上 | 除非是数学或真实矩阵结构 |
| 打法 | 军事化用语 | 说清楚具体的策略 |
| 颗粒度 | 技术黑话的滥用 | "细节程度"或具体说明 |
| 降维打击 | 同上 | 说清楚具体的竞争优势 |
| 卡位 | 同上 | 说清楚具体的市场定位 |
| 护城河 | 除引用语境外禁止 | 说清楚具体的竞争壁垒 |

## 五、空洞修饰词（无数据时禁止）

这些词本身不是问题——**问题是没有数据支撑时使用它们**。

| 词汇 | 有数据时 | 无数据时 |
|------|----------|----------|
| 显著 | "显著提升 40%" ✅ | "显著提升" ❌ → 给出具体数字 |
| 大幅 | "大幅缩短至原来的 1/10" ✅ | "大幅缩短" ❌ → 给出时间对比 |
| 快速 | "3 分钟内完成（此前需要 2 小时）" ✅ | "快速完成" ❌ → 给出时间 |
| 高效 | "效率提升 3 倍" ✅ | "更加高效" ❌ → 给出效率指标 |
| 强大 | 不推荐使用，即使有数据也建议换成具体描述 | ❌ |

## 六、AI 味词汇

AI 生成文本的高频特征词。出现 2 个以上即判定有 AI 味。

| 词汇 | 替代 |
|------|------|
| 不得不说 | 删除，直接说 |
| 有一说一 | 删除 |
| 毋庸置疑 | 删除，如果真的毋庸置疑就不需要说 |
| 不言而喻 | 同上 |
| 不可否认 | 同上 |
| 值得一提的是 | 直接说 |
| 首先……其次……最后…… | 改为无序列表或自解释标题 |
| 一方面……另一方面…… | 用表格或分节替代 |
| 总的来说 | 删除 |
| 需要指出的是 | 直接指出 |

## 七、英文常见问题（评价英文文章时使用）

| 问题 | 示例 | 替代 |
|------|------|------|
| Overused verbs | leverage, utilize, harness | use, apply, build with |
| Buzzwords | cutting-edge, game-changing, revolutionary | 用具体描述替代 |
| Hedge words | somewhat, relatively, arguably | 删除或量化 |
| Filler phrases | It is worth noting that, It should be noted | 直接说内容 |
| Passive voice overuse | "was implemented by the team" | "the team implemented" |
